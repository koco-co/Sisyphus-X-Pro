## 📋 测试流程

### 标准测试流程

每位 e2e-tester 遵循：

1. **启动测试环境**:
   ```bash
   cd frontend && npm run dev
   # 访问 http://localhost:3000
   # 需要先登录（开发模式可免登录）
   ```

2. **编写/运行 E2E 测试**:
   - 使用 Playwright 编写测试用例
   - 使用 Chrome DevTools MCP 工具进行浏览器自动化
   - 覆盖所有分配的功能点

3. **记录测试结果**:
   - ✅ 通过的功能
   - ❌ 失败的功能
   - ⚠️ 发现的新 Bug
   - 📸 截图和视频证据

4. **报告问题**:
   - 发现任何问题立即报告给 team-lead
   - 详细描述复现步骤
   - 附上截图或视频证据
